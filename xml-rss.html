XML Syntax Rules


6 important rules that must be adhered to in order to create a valid XML file:

Rule 1: XML Elements need to be nested correctly


Any elements in an XML document must be nested within each other where applicable.


Code sample: <p> <b> This paragraph is in bold </b> </p>

Rule 2: XML attributes need to be quoted.


XML attributes work the same as in HTML, however they always require quotation marks.


Code sample: <div class="topic-1">

Rule 3: Correct format must be used when writing comments


Comments are written the same way as in HTML. the comment must start with "<!--" and end with "-->". A comment cannot have two dashes in the middle.


Code sample: <!--This is a comment-->

Rule 4: White-space will be kept in XML


Unlike HTML, XML will not shorten multiple spaces into a single space.


Code sample: <p>Helloㅤㅤㅤworld!</p> will display as "Helloㅤㅤㅤworld!"

Rule 5: XML tags are case sensitive


Caution should be used when writing an XML file as the recognised tag can differ between upper and lower case


Code sample: <Letter> is a different tag to <letter>

Rule 6: Entity references must be used


Using a "<" symbol, for example, within an element will create an error as the parser will think you are creating a new tag. Instead you must use an entity reference such as "&lt;".


Code sample: Will work: "<p>$30 &lt; $200</p>". This will create an error: "<p>$30 < $200</p>".
